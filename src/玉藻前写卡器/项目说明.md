# 一键角色卡写卡器

## 项目定位

这是一个酒馆助手后台脚本项目。脚本通过按钮打开一个覆盖酒馆页面的独立 iframe 面板，用户只填写世界观、角色素材和卡面图片；脚本把酒馆当后台 API 使用，自己组装内置的秋青子写卡预设壳，完成生成、写入和绑定。

## 核心边界

- 不是新建一套预设，也不是调用酒馆当前启用预设；而是把 `src/秋青子写卡预设/秋青子写卡预设.json` 适配成脚本内置的组装预设。
- `【一般条目】` 到 `【/MVU条目】` 整段不作为固定外壳复刻；这段知识只在对应任务调用时按需内置。
- 上述区间之外的原预设条目按原顺序、原启用状态、原角色身份复刻到 `generateRaw({ ordered_prompts })`。
- 用户模块输入插入原预设的 `🔓 继续-Kemini` 位置，即替换其中的 `{{lastUserMessage}}`。
- 原预设中的酒馆 builtin 占位（角色定义、世界书、聊天历史等）不传入 `ordered_prompts`；生成时只使用脚本组装的提示词、用户模块输入和任务知识。
- 思维链段保留原标签结构和位置，但适配为简短任务自检，不要求模型输出完整隐藏推理；前端解析时只取 `<content>` 内的成品。
- 只学习 `src/秋青子写卡预设/伪IDE/index.ts` 的打开方式：脚本按钮、全屏 iframe、Vue 挂载、样式隔离、关闭卸载。
- 不复用伪 IDE 前端页面，不使用 `src/wtc`。

## 一键流程

1. 用户先在酒馆创建或打开目标角色卡。
2. 点击脚本按钮打开写卡器。
3. 输入世界书名称，脚本创建或确认世界书，并绑定为当前角色主世界书。
4. 用户填写世界观、角色1素材；可新增角色2、角色3等。
5. 第一次 AI 调用：按原预设适配壳 + 世界观知识，生成世界观条目并写入世界书。
6. 世界观生成完立即写入绑定世界书。
7. 按角色顺序逐次 AI 调用：按原预设适配壳 + 角色人设/调色盘知识，生成角色基础、调色盘、二次解释和 EJS 多阶段中文人设；每个角色生成完立即写入绑定世界书，并刷新角色速览。
8. 前端按世界书配置规则写入：
   - 世界观：蓝灯，角色定义前，order 1。
   - 角色速览：蓝灯，角色定义前，order 4。
   - 单角色卡：角色详细条目蓝灯，角色定义后。
   - 多角色卡：角色速览蓝灯，角色详细条目绿灯。
   - 二次解释：D0 system。
   - EJS 多阶段人设：角色定义后，order 99。
   - 所有写入条目禁止递归、防止进一步递归。
9. 固定生成 MVU：
   - MVU 运行脚本：把外部 `酒馆助手脚本-MVU.json` 写入当前角色卡脚本库并启用。
   - 变量结构脚本：写入当前角色卡脚本。
   - 初始变量：世界.当前时间、世界.当前地点、每个角色.好感度。
   - 变量更新规则。
   - 变量列表。
   - 变量输出格式。
   - 变量输出格式强调。
10. 后 4 个外部正则写入当前角色局部正则并启用；脚本另生成状态栏显示正则与占位符隐藏正则，按角色数量接入 `stat_data.*`。
11. 用户导入图片时，将图片写入当前角色卡头像。

## 文件规划

- `index.ts`：脚本入口和全屏 iframe 挂载。
- `App.vue`：一键写卡面板。
- `types.ts`：内部数据类型。
- `utils/presetAdapter.ts`：秋青子原预设结构适配器。
- `utils/generation.ts`：世界观和角色生成调用。
- `utils/parser.ts`：解析 AI 输出。
- `utils/artifacts.ts`：标签、MVU、EJS、状态栏和世界书条目生成。
- `utils/worldbook.ts`：世界书创建、绑定、写入。
- `utils/character.ts`：头像和角色脚本写入。
- `utils/regex.ts`：外部 MVU 正则导入、状态栏显示/隐藏正则写入。
