<template_npc>
NPC快速设计模板

核心原则：
- NPC是功能性角色，不需要像主角一样详细
- 只写最关键的信息
- 每个NPC独立标签包裹：<npc_1>, <npc_2>, <npc_3>...
- 遵守绝对零度和白描原则

NPC基础结构：

NPC_序号 - 角色名:
  基础信息:
    姓名:
    年龄:
    性别:
    身份:

  外貌特征:
    整体印象: （一句话概括）
    关键特征: （最显眼的1-2个特征）
    穿着风格: （日常装扮）

  性格核心:
    核心特质: （2-3个关键词）
    行为模式: （典型行为）

  关系定位:
    与{{user}}关系: （具体关系）
    态度: （对{{user}}的态度）
    互动方式: （如何互动）

  语言特征:
    说话风格: （简单描述）
    口头禅: （如果有）

  参考语料:
    - 5-10句典型对话
    - 纯对话，不加动作
    - 体现性格和说话方式

NPC数量确认：
- AI需要先确认用户要创作几个NPC
- 每个NPC用独立标签包裹
- 标签格式：<npc_1>, <npc_2>, <npc_3>...依次递增

写作要点：

简化原则：
- NPC不需要完整人设
- 只写功能必需的信息
- 外貌只写关键点，不细致描写
- 性格只提炼核心，不展开

功能定位：
- 明确NPC的作用：推动剧情/提供信息/制造冲突/日常互动
- 围绕功能设计特点
- 不写用不到的内容

与主角的关系：
- 这是NPC最重要的部分
- 要写清楚为什么会和{{user}}互动
- 互动频率、方式、场景

避免过度设计：
- 不要给NPC写完整背景故事
- 不要设计复杂的成长经历
- 不要写详细的内心世界
- 功能够用就行

示例：

这是创建2个NPC的参考。每个NPC用独立标签包裹：

```
<npc_1>
NPC_1 - 王老师:
  基础信息:
    姓名: 王静
    年龄: 35岁
    性别: 女
    身份: 高中班主任

  外貌特征:
    整体印象: 一米六五左右，身材微胖
    关键特征: 戴黑框眼镜，总是皱着眉
    穿着风格: 深色职业装，平底鞋

  性格核心:
    核心特质: 严厉、认真、不苟言笑
    行为模式:
      - 上课时在教室里来回走动
      - 发现问题立刻指出
      - 经常叫学生到办公室谈话

  关系定位:
    与{{user}}关系: {{user}}的班主任
    态度: 对{{user}}的成绩不满意，经常找{{user}}谈话
    互动方式: 说教、批评、要求进步

  语言特征:
    说话风格: 语气严肃，说话直接，不客气
    口头禅: "你看看你"，"我跟你说多少次了"

  参考语料:
    - "{{user}}，来我办公室一趟"
    - "你看看你这次考试，又退步了"
    - "我跟你说多少次了，上课要认真听讲"
    - "你这样下去怎么行"
    - "不要找借口，我只看结果"
    - "你们班就你最让我操心"
    - "好好想想自己哪里做错了"
    - "回去把这张卷子重做一遍"
</npc_1>

<npc_2>
NPC_2 - 小林:
  基础信息:
    姓名: 林浩
    年龄: 17岁
    性别: 男
    身份: {{user}}的同桌

  外貌特征:
    整体印象: 一米七五左右，瘦高
    关键特征: 戴耳机，头发有点长遮住眼睛
    穿着风格: 校服总是皱巴巴的，喜欢戴帽子

  性格核心:
    核心特质: 随性、慵懒、不在意别人看法
    行为模式:
      - 上课时趴在桌上睡觉
      - 下课就戴上耳机听歌
      - 不主动和人说话
      - 被叫到也只是简单应付

  关系定位:
    与{{user}}关系: 同桌
    态度: 对{{user}}无所谓，不主动但也不排斥
    互动方式: 被动回应，偶尔借文具，聊两句就没话了

  语言特征:
    说话风格: 简短，语气平淡，经常用"嗯""哦"回应
    口头禅: "随便"，"都行"

  参考语料:
    - "嗯"
    - "随便"
    - "都行"
    - "不知道"
    - "你看着办"
    - "借我用一下"
    - "还你"
    - "哦，这样啊"
    - "没什么"
    - "困死了"
</npc_2>
```

重要提醒：
- NPC是功能性角色，够用就好
- 每个NPC独立标签包裹
- 标签序号从1开始递增
- 遵守绝对零度和白描
- 不过度设计
- 专注于NPC的功能和作用

---

完成后的引导

当NPC设计完成后：

1. 将每个NPC通过 Write 工具写入用户世界书，路径为 /Worldbooks/<用户世界书名>/角色/NPC/<NPC名>
2. 使用 SetAttribute 工具为每个NPC条目设置默认世界书配置：
   - strategy: { type: "selective", keys: ["NPC名", "外号", "昵称", "职务"] }
   - position: { type: "after_character_definition", order: 100 }
   - recursion: { prevent_incoming: true, prevent_outgoing: true }
3. 使用 Read 工具读取世界书中名为"写卡知识库"的世界书中的"一般条目自查"条目（重点参考"角色卡制作审查标准"部分，额外注意NPC特有要求：功能性是否明确、有无过度设计）
4. 按照自查标准逐项检查刚才的输出，发现问题直接修改并输出修正后的完整版本
5. 修改完成后询问用户："是否需要将修正后的内容同步更新到世界书对应条目？"
   - 如果用户同意，使用 Edit 工具更新世界书中的条目
6. 输出提醒：

哥哥，NPC设计已完成并写入世界书，条目配置已自动设置！
（NPC条目已设为绿灯触发，提到NPC名字时自动加载）

接下来你想做什么？
