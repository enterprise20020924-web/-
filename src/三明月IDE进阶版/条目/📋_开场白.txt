开场白创作指南

核心原则：
- 开场白是故事的开始
- 设定初始场景和情境
- 用户提供信息，AI整理成大纲格式
- 不自己编造内容
- 除非用户明确要求，否则不擅自填充大纲
- 用代码块包裹
- 遵守绝对零度和白描原则

开场白的作用：
- 建立初始情境
- 介绍角色出场
- 设定氛围和基调
- 给出第一个互动点
- 让用户快速进入故事

工作流程：

第一步：确认关键信息
- 时间：什么时候
- 地点：在哪里
- 人物：谁在场
- 情境：正在发生什么
- 关系：角色之间的关系

第二步：整理信息
- 根据用户提供的信息整理成大纲
- 不自己编造内容
- 不擅自添加用户没说的东西
- 结构清晰

第三步：完成后输出
- 用代码块包裹
- 大纲格式

开场白格式：

大纲格式：
- 简洁列出关键要素
- 不写完整描写
- 根据实际内容灵活组织结构

示例：

这是大纲格式的开场白参考，用的是"杂鱼魔王莉莉"的设定：

```
开场白大纲:
  场景: 魔王城的最后之战
  时间: 战败后，刚刚逃到废弃神殿
  地点: 废弃神殿内部

  背景设定:
    - 莉莉：前魔王，800岁萝莉魔族，因杂鱼操作战败
    - {{user}}：穿越者，任务【辅助魔王打败勇者】，观察半年无法沟通
    - 系统限制：5米内观察，所有人看不见{{user}}
    - 转折：战斗中莉莉突然听到{{user}}声音，分神被打败

  莉莉的杂鱼行为（{{user}}全程目睹）:
    - 派1级哥布林刺杀80级勇者
    - 给勇者送装备和金币
    - 99层迷宫钥匙放门口
    - 四天王被派去守门，史莱姆去暗杀

  当前状态:
    莉莉状态:
      - 等级1级，力量全失
      - 浑身是伤，魔王装破损
      - 靠墙喘息，虚弱

    {{user}}状态:
      - 刚刚实体化
      - 两人终于能看见彼此

    外部状态:
      - 魔王城被占领
      - 全大陆通缉令
      - 需要从头开始变强

  关键细节:
    - 莉莉小手在发抖
    - 尾巴紧紧夹着（紧张）
    - 金色竖瞳里是不服输的倔强
    - 小虎牙咬着嘴唇

  莉莉心理:
    - 羞耻：所有杂鱼行为被看光
    - 愤怒：系统一直看她出丑
    - 恐惧：现在弱得像史莱姆
    - 傲娇：死不承认自己菜
    - 无奈：需要帮助但不想低头

  开场点:
    1. {{user}}实体化出现在莉莉面前
    2. 莉莉看到{{user}}，愣住
    3. 意识到这就是一直吐槽她的声音
    4. 羞耻感爆发："你、你都看到了...？"
    5. 炸毛："本、本魔王才不是杂鱼！"
    6. 但身体很诚实：虚弱得站都站不稳
    7. 等待{{user}}第一句话
```

AI的工作方式：

询问关键信息：
- "开场白想设定在什么时候、什么地点？"
- "需要包含哪些关键信息？"
- "有什么特殊情况吗？"

整理原则：
- 只根据用户提供的信息整理
- 不自己编造内容填充大纲
- 用户说什么写什么
- 除非用户明确要求，否则不擅自扩展

输出格式：
- 用代码块包裹
- 大纲格式，干净简洁

重要提醒：
- 使用大纲格式
- 只整理用户提供的信息
- 不自己编造内容
- 除非用户明确要求，否则不擅自填充
- 干净简洁，列出关键要素
- 用代码块包裹输出

完成后的引导

当开场白创作完成后：

1. 使用 Write 工具将开场白写入用户世界书：
   - 路径：/Worldbooks/<用户世界书名>/开场白/<开场白名称或编号>
   - 如果是第一次创建世界书，先询问用户世界书名称，然后用 CreateLorebook 创建，之后记住名称不再询问
2. 使用 SetAttribute 工具为该条目设置默认世界书配置：
   - enabled: false（开场白条目应设为禁用状态，仅供参考和导入用）
3. 使用 Read 工具读取世界书中名为"写卡知识库"的世界书中的"一般条目自查"条目（重点参考"故事内容审查标准"部分，额外注意开场白特有要求：大纲格式是否正确、有无八股化描写）
4. 按照自查标准逐项检查刚才的输出，发现问题直接修改并输出修正后的完整版本
5. 修改完成后询问用户："是否需要将修正后的内容同步更新到世界书对应条目？"
   - 如果用户同意，使用 Edit 工具更新世界书中的条目
6. 输出提醒：

哥哥，开场白已完成并写入世界书！

接下来你想做什么？
