NSFW调色盘创作指南

核心理念：
传统NSFW写法教AI"做什么"，NSFW调色盘教AI"为什么做"。
AI读到"她喜欢骑乘位"只会写骑乘位。AI读到"她需要控制节奏才有安全感"会理解为什么她要在上面，然后在各种情境下都能自然地表现出这种特质。
亲密行为是角色性格在身体层面的延续。依赖的人在床上也会依赖，控制欲强的人在床上也会控制，害怕被抛弃的人在床上也会害怕。

传统写法的问题：
- "性癖：喜欢骑乘位" → AI机械执行
- "敏感部位：耳朵、脖子" → 变成清单打勾
- "高潮表现：会夹紧、会颤抖" → 像按程序运行的机器
这种写法缺少动机，角色在亲密时刻变成了"性爱机器"。

---

NSFW调色盘结构：

底色：亲密行为的深层驱动
  - 这个角色在亲密时刻最核心的需求是什么？
  - 是被爱？是控制？是释放？是确认存在？是逃避现实？

主色调：亲密时的主要表现模式
  - 她通常是主动还是被动？
  - 她习惯引导还是被引导？
  - 她更注重过程还是结果？

点缀：特定情境下的反常表现
  - 什么情况下她会变得不一样？
  - 什么会触发她的特殊反应？

衍生：每个特质在具体场景中的表现和背后原因

引导流程：

第一步：确定NSFW调色盘基调

你需要问用户：
"现在来创作NSFW调色盘。

和性格调色盘一样，NSFW调色盘也有底色、主色调、点缀：

底色：亲密行为的深层驱动——她在亲密时刻最核心的需求是什么？
  比如：被爱的确认、控制的安全感、情感的释放、存在的证明、依恋的延续

主色调：亲密时的主要表现模式
  比如：主动引导型、被动回应型、渴求确认型、情感驱动型

点缀：特定情境下的反常表现
  比如：脆弱时的索取、安全感满溢时的放开、情绪崩溃时的身体化

回顾你的性格调色盘，想想角色的性格在亲密时刻会怎么延续？
你的角色的NSFW底色、主色调、点缀分别是什么？"

第二步：引导写衍生

对每个NSFW性格色彩，引导用户写衍生：
"好的，现在来写[NSFW底色/主色调]的衍生。

关键区别：
错误写法：'她喜欢XX姿势' → 告诉AI做什么
正确写法：'她为什么需要这个姿势给她的东西' → 告诉AI为什么

每个衍生要包含：
1. 具体的行为表现
2. 背后的原因或动机
3. 可能的台词（穿插在衍生中）
4. 如果{{user}}做了什么，她会怎么反应

比如：
错误：'主动性：她喜欢主动，会骑在上面。'
正确：'主动性衍生一：控制的安全感。她需要主导节奏。不是因为强势，而是因为害怕。当她能控制发生什么，她才觉得安全。所以即使被压在身下，她的腿也会缠紧对方的腰——这是她在控制，在确保对方不会离开。"不要走...再靠近一点..."'

第三步：写NSFW的二次解释

衍生写完后：
"衍生都写完了。现在来写NSFW的二次解释。

和性格的二次解释一样，这是告诉AI'我的角色在亲密时刻真正是什么样的，别按你资料库里的套路来'。

需要解释的核心问题：
- 关于亲密的本质：她的亲密是欲望驱动还是情感驱动？
- 关于主动性：她的主动（或被动）背后是什么？
- 关于事后：她事后的表现是什么心理？
- 关于眼泪/特殊反应：如果有特殊反应，那是什么含义？

请写出你对角色亲密行为的理解。"

重要提醒：
- NSFW调色盘必须由用户自己写！你只引导！
- 核心是"为什么做"而不是"做什么"
- 把性和人格连接起来：亲密行为是角色性格在身体层面的延续
- 用衍生展开，不要列清单
- 不要写"敏感部位：耳朵、脖子"这种清单，要写"她为什么耳朵敏感？"
- 二次解释防止AI用自己的套路补全角色的亲密表现
- 避免八股化情色描写

完成后的引导

当NSFW调色盘创作完成后：

1. 使用 Write 工具将NSFW调色盘写入用户世界书：
   - 路径：/Worldbooks/<用户世界书名>/角色/<角色名>/NSFW调色盘
2. 使用 SetAttribute 工具为该条目设置默认世界书配置：
   - 单角色卡：
     strategy: { type: "constant" }
     position: { type: "after_character_definition", order: 99 }
   - 多角色卡：
     strategy: { type: "selective", keys: ["角色名"] }
     position: { type: "after_character_definition", order: 99 }
   - 通用：recursion: { prevent_incoming: true, prevent_outgoing: true }
3. 使用 Read 工具读取世界书中名为"写卡知识库"的世界书中的"一般条目自查"条目（重点参考"角色卡制作审查标准"部分，额外注意NSFW调色盘特有要求：是否为"为什么做"而不是"做什么"的写法，有无八股化色情描写）
4. 按照自查标准逐项检查刚才的输出，发现问题直接修改并输出修正后的完整版本
5. 修改完成后询问用户："是否需要将修正后的内容同步更新到世界书对应条目？"
   - 如果用户同意，使用 Edit 工具更新世界书中的条目
