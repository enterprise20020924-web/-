## 1. 项目目标

在“明月秋青”的标准模式中实现类似主流 Agent 的计划模式 / To-do 系统。

核心目标是让 AI 不只依赖聊天记录记住任务, 而是在每次生成时都能读取当前 active plan。用户可以用简短输入启动计划, AI 负责起草计划和建议更新, 前端负责确认、保存、注入和归档。

v1 只覆盖标准模式和普通聊天入口。宝宝辅食模式、MVU 剧情任务系统、世界书长期项目档案导出均不纳入 v1。

## 4. 设计决策记录

| 决策点 | 选择 | 原因 |
|---|---|---|
| 计划存储 | 当前聊天变量 | 避免跨聊天串味, 与现有聊天收纳机制一致 |
| 变量 key | `明月秋青_计划模式_v1` | 明确命名空间, 便于后续迁移 |
| prompt 注入 | `injectPrompts` | 覆盖 `/send`、`/trigger`、`/continue`, 不污染聊天记录 |
| AI 权限 | 只建议, 不自动落盘 | 防止 AI 误改 todo 或擅自结束计划 |
| 完成判定 | AI 建议 + 用户确认 | 避免单靠 todo 勾选造成误关闭 |
| 静态条目 | 长期保留 | 条目是规则模板, 完成计划只清 active plan |
| v1 范围 | 标准模式优先 | 宝宝辅食流程更复杂, 后续单独接入 |

## 5. 计划状态机

计划状态固定为:

```ts
type PlanStatus =
  | 'draft'
  | 'active'
  | 'paused'
  | 'ready_to_complete'
  | 'completed'
  | 'archived';
```

Todo 状态固定为:

```ts
type TodoStatus = 'todo' | 'doing' | 'blocked' | 'done' | 'skipped';
```

生命周期:

```text
无计划
-> AI 生成 PlanDraft
-> 用户确认采用
-> active
-> AI 生成 PlanUpdate 建议
-> 用户确认应用或拒绝
-> AI 可建议 ready_to_complete
-> 用户确认完成
-> completed + archive
-> activePlanId 清空, prompt 不再注入计划
```

约束:

| 约束 | 状态 |
|---|---|
| AI 不能自动激活计划 | 待实施 |
| AI 不能自动应用 PlanUpdate | 待实施 |
| AI 不能自动完成计划 | 待实施 |
| 用户确认后才归档 | 待实施 |
| 计划 paused 时不注入 prompt | 待实施 |
